# Travel Wallet Telegram Bot

Telegram-бот на Python для учёта бюджета в путешествиях.

Бот помогает:
- создавать несколько путешествий;
- хранить баланс в домашней и местной валюте;
- пересчитывать суммы через `api.exchangerate.host`;
- учитывать расходы;
- смотреть историю расходов с датой расхода и датой курса;
- переключаться между путешествиями;
- удалять ненужные путешествия.

## Возможности

- Создание путешествия по стране отправления и стране назначения.
- Автоопределение валюты:
  - по названию страны;
  - по 3-буквенному коду валюты (`EUR`, `USD`, `RUB` и т. д.).
- Проверка курса через `api.exchangerate.host`.
- Возможность принять курс API или ввести курс вручную.
- Стартовый баланс сразу в двух валютах.
- Учёт расходов в валюте страны пребывания.
- Подтверждение расхода через inline-кнопки.
- История расходов:
  - сумма расхода;
  - дата расхода;
  - дата курса;
  - курс, который использовался.
- Удаление путешествий вместе с расходами.
- Локальное хранение данных в SQLite.

## Используемый API

Проект использует `api.exchangerate.host`.

Основные endpoint'ы:
- `/list` — список поддерживаемых валют;
- `/convert` — конвертация суммы и получение данных по курсу.

Для работы нужен `access_key`.

## Требования

- Telegram Bot Token
- Ключ `api.exchangerate.host`

## Как пользоваться

### 1. Запустить бота

В Telegram нажмите `Start` или отправьте:

```text
/start
```

### 2. Создать новое путешествие

Через меню:
- `Создать новое путешествие`

Или командой:

```text
/newtrip
```

Дальше бот попросит:
1. страну отправления;
2. страну назначения;
3. подтвердить курс или ввести вручную;
4. стартовую сумму в домашней валюте.

После этого создаётся путешествие с балансом в двух валютах.

### 3. Добавить расход

Есть два способа:

#### Способ A. Через отдельную кнопку

- Нажмите `Добавить расход`
- Введите сумму расхода в валюте страны пребывания
- Подтвердите `✅ Да`

#### Способ B. Просто отправить число

Если активно путешествие, можно просто отправить сообщение:

```text
100
```

Бот пересчитает сумму и предложит:

```text
100 CNY = 1280 RUB

Учесть как расход?
```

После подтверждения расход будет сохранён, а баланс уменьшится.

### 4. Посмотреть баланс

Через меню:
- `Баланс`

Или командой:

```text
/balance
```

### 5. Посмотреть историю расходов

Через меню:
- `История расходов`

Или командой:

```text
/history
```

В истории бот показывает:
- сумму расхода;
- эквивалент в домашней валюте;
- дату расхода;
- дату курса;
- курс, который использовался.

### 6. Переключить активное путешествие

Через меню:
- `Мои путешествия`

Или командой:

```text
/switch
```

### 7. Изменить курс вручную

Через меню:
- `Изменить курс`

Или командой:

```text
/setrate
```

### 8. Удалить путешествие

Через меню:
- `Удалить путешествие`

Или командой:

```text
/deletetrip
```

Удаляются:
- само путешествие;
- все связанные расходы.

## Поддерживаемые команды

- `/start` — старт и главное меню
- `/newtrip` — создать новое путешествие
- `/switch` — выбрать активное путешествие
- `/balance` — показать баланс
- `/history` — показать историю расходов
- `/setrate` — изменить курс
- `/expense` — добавить расход
- `/deletetrip` — удалить путешествие

## Структура проекта

- `travel_bot.py` — основная логика Telegram-бота
- `current_api.py` — работа с `api.exchangerate.host`
- `country_currencies.py` — словарь стран и валют
- `wallet_db.py` — SQLite-слой
- `travel_wallet.sqlite3` — база данных
- `requirements.txt` — зависимости
- `.env` — конфигурация

## Хранение данных

Используется SQLite.

В базе сохраняются:
- путешествия;
- текущий активный выбор пользователя;
- история расходов;
- дата расхода;
- дата курса и курс, использованный при пересчёте.

## Обработка ошибок

Бот старается дружелюбно обрабатывать:
- неверный ввод суммы;
- неизвестную страну или валюту;
- отсутствие активного путешествия;
- ошибки API;
- недоступность сети;
- попытку списать больше остатка.

Если API недоступен во время учёта расхода, используется fallback через сохранённый курс путешествия.

## Примечания

- У одного пользователя может быть несколько путешествий.
- У каждого пользователя свой собственный набор данных.
- Для корректной работы бот должен быть постоянно запущен.
- В проекте могут быть временные `DEBUG`-сообщения в консоли для диагностики.

## Пример сценария

1. Создаёте путешествие: `Россия -> Испания`
2. Указываете стартовую сумму: `100000 RUB`
3. Бот создаёт баланс, например:

```text
Остаток: 1000 EUR = 100 000 RUB
```

4. В поездке отправляете:

```text
25
```

5. Бот отвечает:

```text
25 EUR = 2 500 RUB

Учесть как расход?
```

6. После подтверждения баланс уменьшается, а запись попадает в историю.
